- 이 자료는 아쉽게 backpressure 에 대해 설명이 부족
- backpressure : 데이터를 막 PUSH 하면 subscriber가 뻑나서 데이터 처리 못하거나 유실되거나 쌓여있어서 지연되거나 일이 발생
  - 그럼 이게 Non blocking 하다고 볼수 없기에 backpressure가 중요한것. pull/push -> 데이터를 몇개만 달라고 요청하는 행위(조절)
